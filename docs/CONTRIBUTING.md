# Contributing to SIEVER Mail

Thank you for your interest in SIEVER Mail.

## External contributions are not currently accepted

The project is in an active stabilisation phase and the maintainer is not yet
ready to take in pull requests from outside collaborators. **Please do not
open PRs against `main`.** Any PR opened during this phase will be closed
without review to keep the queue tidy.

This policy will be revisited once the 1.x baseline ships; this file will be
updated when the project opens to external contributions.

## How you can help today

Two channels are open and very much appreciated:

### 1. GitHub Issues

If you find a bug, a regression, a usability problem or have a feature
suggestion, **open an issue** on the GitHub tracker:

- Use a clear, single-purpose title.
- Describe what you expected vs what actually happened.
- Attach the application version, OS / architecture and steps to reproduce.
- For UI issues, a screenshot or short screen recording is gold.

### 2. Direct contact

For everything that does not fit a public issue (security reports,
private feedback, deployment questions, partnership inquiries) write to
the maintainer at:

📧 **[beltromatti@gmail.com](mailto:beltromatti@gmail.com)**

Security-sensitive reports will always be acknowledged within a reasonable
window and handled responsibly before any public disclosure.

## Code of conduct

Be civil, be precise, be kind. The project is open-source software written
in spare time; constructive feedback is welcome, hostility is not.

## License

By interacting with the project (filing issues, sending emails or, in the
future, opening pull requests) you agree that any code you contribute will
be released under the same Apache License 2.0 that covers the rest of the
project.
